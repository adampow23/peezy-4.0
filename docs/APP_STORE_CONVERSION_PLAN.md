# App Store Conversion Plan: Peezy

This plan focuses on download conversion, not broad brand marketing. It assumes the primary market is U.S. iPhone users planning a residential move.

## Core Conversion Thesis

Peezy should not be positioned as a generic checklist. The high-converting promise is:

> A personal moving concierge that builds your custom move plan and helps you handle the stressful parts one calm task at a time.

The product page should make three things obvious within the first viewport:

1. Peezy creates a personalized plan from your actual move details.
2. Peezy reduces the move into simple daily actions instead of a giant checklist.
3. Peezy helps with real moving logistics: movers, utilities, mail, inventory, records, and support.

## Highest Priority Fixes

1. Replace broad attitude-led first impression with outcome-led App Store copy.
   - Website line like "hell of a lot easier" is memorable, but the App Store first impression should feel more trusted and broadly family-safe.
   - Recommended first-view language: "Your move, mapped out" or "Moving, handled daily."

2. Make the first screenshot the personalized plan reveal.
   - The strongest aha moment is not a welcome screen. It is the moment Peezy says it built a plan and shows a real task count.
   - Use the Summary screen or a polished App Store capture with a realistic task count.

3. Test the icon.
   - Current icon: black background with white `PZY`.
   - Strength: premium, stark, memorable.
   - Risk: low semantic clarity in search results. A cold user may not connect `PZY` to moving help.
   - Test against a moving-specific mark using the same premium palette: `P` plus route/check/box motif, or a yellow-accent variant.

4. Keep subscription language accurate.
   - App Store description should not include exact prices because pricing is already displayed and can vary by region.
   - If using "free," tie it only to the actual introductory trial or free starting experience. Avoid implying the full service is free.

## Recommended Metadata Package

### App Name

Primary:

`Peezy: Moving Concierge`  
Characters: 23/30  
Why: Uses the strongest category-positioning phrase while keeping the brand first.

Alternatives:

- `Peezy: Moving Checklist` - 23/30. More searchable, less differentiated.
- `Peezy: Moving Planner` - 21/30. Clearer than checklist, but less premium than concierge.
- `Peezy - Moving Made Easy` - 24/30. Strong conversion phrase, weaker exact category targeting.

### Subtitle

Primary:

`Custom plan. Calm daily tasks`  
Characters: 29/30  
Why: Communicates personalization and the daily-action model without repeating title terms.

Alternatives:

- `Plan, book, move calmly` - 23/30. Better if vendor coordination is the main sell.
- `One calm task at a time` - 23/30. Best emotional clarity, lower keyword density.
- `Your move, handled daily` - 24/30. Strongest if "handled" is legally and operationally accurate.

### Keyword Field

Notes:

- Do not include competitor app names.
- Do not include `app`, category names, or pricing words.
- Validate the final keyword set with actual App Store keyword tools before shipping.

## Description Draft

Moving has too many hidden tasks. Peezy finds them, builds your personalized plan, and helps you handle one calm step each day.

Peezy is a personal moving concierge for people who have better things to do than move. Answer a short assessment about your move, then get a custom plan built around your timeline, home, household, services, and moving style.

What Peezy helps with:

- Personalized moving plan built from your details
- Daily task stream so the move feels manageable
- Movers, cleaners, truck rental, utilities, mail, and address updates
- AI inventory scanning for room-by-room move prep
- Vetted service help without paid-listing noise
- Support chat when plans change or details get messy
- Full task timeline for everything still to do

How it works:

1. Take a quick moving assessment.
2. Peezy builds your custom plan.
3. Open the app each day and handle the next right thing.

Peezy is not a generic task manager. It is built for the actual chaos of moving: deadlines, vendors, records, utilities, packing, cleaning, kids, pets, vehicles, and all the little things that are easy to miss.

Start with your plan. Move with less mental load.

| Slot | Caption | Screen To Show | Conversion Job |
|---|---|---|---|
| 1 | Your move, mapped out | Personalized Moving Plan summary with task count | Show the aha moment immediately |
| 2 | One task at a time | Home daily task card | Reduce anxiety and show simplicity |

| 5 | Find movers without guessing | Vendor or mover workflow | Highlight trust and concierge value |

| 9 | Know what is done | Done / In Progress task tabs | Show progress and control |

## Screenshot Design Brief

- Background: lightBase with subtle ice blue and lavender brand atmosphere.
- Text: dark ink, large, sentence case, 4-6 words per screenshot.
- Device: straight iPhone frame or full-bleed app UI. Avoid busy angled mockups for the default page.
- Content: use real move examples, not placeholder copy.
- Avoid: login screens, empty states, legal/pricing screens, and screenshots where text is too small to read in search.

## Product Page Optimization Tests

Apple Product Page Optimization can test app icons, screenshots, and app previews with up to three treatments. Tests run up to 90 days and need enough impressions to reach confidence.

### Test 1: First Screenshot Hook

Hypothesis: If the first screenshot shows the personalized plan reveal instead of a generic app screen, conversion rate will improve because users instantly understand the payoff.

Variants:

- Control A: Current first screenshot.
- Variant B: `Your move, mapped out` with personalized plan/task count.
- Variant C: `One task at a time` with Home daily task card.

Success metric: Product page conversion rate.

### Test 2: Icon Clarity

Hypothesis: If the icon keeps Peezy's premium feel but adds a moving-related symbol, search tap-through and download conversion will improve because the app category becomes clearer before users read the subtitle.

Variants:

- Control A: Current black `PZY` icon.
- Variant B: Black/deep-ink icon with yellow-accent `P` plus route/check motif.
- Variant C: Light glass/yellow moving-box mark with small `P` or `PZ`.

Success metric: Conversion rate, with search traffic watched separately.

### Test 3: Trust Versus Utility In Slot 3

Hypothesis: Users who are anxious about vendors may convert better when trust appears early; users who want control may convert better when task depth appears early.

Variants:

- Control A: Assessment personalization in slot 3.
- Variant B: Vetted movers/vendor trust in slot 3.
- Variant C: Full task timeline in slot 3.

Success metric: Conversion rate by source type and keyword cluster.

## Custom Product Pages

Apple currently supports up to 70 additional custom product pages. Use these for distinct traffic sources or audience intents, not as a substitute for random A/B testing.

Recommended pages:

| Page | Audience / Traffic | First Screenshot Hook |
|---|---|---|
| Apartment move | Renters, apartment searches | Elevators, keys, deposits, utilities |
| Family move | Parents | Schools, daycare, doctors, pets |
| Long-distance move | Cross-state search/ad traffic | Movers, vehicles, timelines, records |
| DIY move | Truck rental and packing traffic | Truck, supplies, inventory, cleaning |
| Vetted services | Mover/cleaner referral traffic | "No paid listings. Vetted help." |
| Inventory | Storage/packing traffic | Room scanning and estimate prep |

Measure each page on product page views, downloads, conversion rate, trial starts, and subscription value. A lower-download page can still win if it brings better subscribers.

## Baseline Metrics To Pull

Before changing assets, capture a 28-day baseline in App Store Connect:

- Unique impressions
- Product page views
- First-time downloads
- Conversion rate
- Source type: App Store Search, Browse, Referrals, Web
- Device split: iPhone vs iPad if available
- Territory split, especially U.S.
- Trial starts and paid conversion by source if available

Apple defines App Store conversion rate as total downloads and pre-orders divided by unique device impressions. For product page tests, the main performance measure is the estimated percentage of users who download or pre-order after viewing the page variant.

## 30-Day Execution Order

Week 1:

- Confirm current App Store title, subtitle, keywords, category, screenshots, icon, and baseline metrics.
- Capture clean in-app screenshots for the 10-slot plan.
- Replace any App Store first-view profanity with outcome-led copy.

Week 2:

- Ship updated metadata and default screenshots if the app is not yet live or if a new version is already planned.
- Prepare icon variants and screenshot variants for PPO.

Week 3:

- Launch PPO Test 1 on the first screenshot.
- Build the first two custom product pages: Apartment Move and Long-Distance Move.

Week 4:

- Review early directional data without stopping tests prematurely.
- Prepare Test 2 icon variants.
- Expand custom product pages only for traffic sources you can actually route.

## Decisions To Confirm

- Primary App Store category: Lifestyle vs Productivity.
- Whether "handled" is acceptable legally/operationally, or whether "guided" and "helped" should be used more often.
- Whether vetted vendors are available broadly enough to lead with that promise in screenshots.
- Current App Store listing and App ID.
- Current product page conversion rate and daily impressions.
